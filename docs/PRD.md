# 🧶 뜨개질/코바늘 도안 제작 SaaS 제품 기획서 (PRD)

## 1. 프로젝트 개요 (Overview)
- **제품명 (가칭):** KnitCanvas / StitchCraft
- **한 줄 소개:** "그리면 도안이 되고, 쓰면 완성되는 마법 같은 전문 뜨개 편집 툴"
- **목표:** 기존에 일러스트레이터, 엑셀, 수작업에 의존하던 비효율적인 뜨개 도안 제작 프로세스를 웹/태블릿 기반의 맞춤형 툴로 대체하여, 작업 시간을 단축하고 퀄리티를 상향 평준화함.

## 2. 타겟 고객 및 문제 정의 (Target Audience & Pain Points)

### 2.1 메인 페르소나
- **전문 니트 디자이너 / 유료 도안 판매자** (Ravelry, Etsy, 스마트스토어 등)
- **뜨개/코바늘 공방 운영자 및 클래스 강사**
- **자신만의 창작 도안을 문서화하고 싶은 취미/아마추어 크리에이터**

### 2.2 해결하고자 하는 문제 (Pain Points)
1. **불편하고 무거운 범용 툴:** 어도비 일러스트레이터로 기호를 하나씩 복사/붙여넣기하며 줄을 맞추는 등 극심한 노가다 소요.
2. **이중 작업의 고통:** 기호 차트(Chart)를 완성한 후, 서술형(Written) 도안을 작성하기 위해 또다시 일일이 타이핑 및 검수 필요.
3. **복잡한 사이즈 그레이딩 (Grading):** 다양한 코수/단계를 S, M, L 등의 사이즈별로 계산할 때 실수 발생이 잦고 많은 시간 소요.
4. **저작권 보호의 어려움:** 불법 공유를 막기 위한 구매자별 워터마크 처리를 수동으로 해야 함.

## 5. UI / UX 화면 가이드라인 (에디터 레이아웃 기준)

웹 에디터 특성상 **Figma**나 **Notion**같은 생산성 툴의 직관적인 배치를 따릅니다.

1. **Top Bar (상단 바):** 도안 제목, 실행 취소/재실행, Auto-save 동기화 상태 표시, **[내보내기]** 버튼 배치.
2. **Left Sidebar (좌측 패널):** 
   - 탭 메뉴: 코바늘 / 대바늘 / 저장된 커스텀 기호.
   - 텍스트 검색창으로 기호 빠른 검색 (예: "팝콘뜨기").
3. **Center Canvas (중앙 작업 공간):** 
   - 마우스 휠 또는 멀티 터치를 활용한 직관적인 Zoom In / Zoom Out.
   - 단수(Rows) 및 콧수(Stitches) 눈금자 제공.
4. **Right Sidebar (우측 패널):**
   - 위쪽 구역: 선택한 기호 속성 조절 (회전각, 크기, 색상 등).
   - 아래쪽 구역: **Written View (서술형 뷰어)**. 캔버스에 추가한 기호 순서대로 텍스트가 실시간 연동되어 출력.

## 6. 추천 기술 및 아키텍처 스택 (Tech Stack)

MVP를 가장 빠르고 사용자 경험 훼손 없이 구축할 조합입니다.

- **프론트엔드 (웹 애플리케이션):** 
  - `Next.js` (SEO 최적화 랜딩페이지 및 App Router 활용 편의성)
  - `TypeScript` (복잡한 캔버스의 좌표, 기호 객체들의 타입 안정성 확보)
  - `Tailwind CSS` (컴포넌트 스타일링)
- **에디터/캔버스 엔진:** 
  - 🥇 **[`Konva.js` (React-Konva)](https://konvajs.org/):** 웹 캔버스 구현에 가장 널리 쓰이며, 수백 개의 렌더링 노드를 관리, 드래그/스냅/이벤트 처리에 최적화.
- **백엔드 리소스 / DB (Auth & DB):**
  - `Supabase` (BaaS 형태, 빠른 소셜 로그인 구현, PostgreSQL을 통한 JSON 형태의 복잡한 도안 데이터 저장 용이)
- **보안:**
  - `DOMPurify` (SVG 업로드 Sanitization)
  - Supabase RLS (Row Level Security) 정책으로 사용자 간 데이터 격리
  - `helmet` 또는 Next.js 미들웨어를 통한 보안 헤더 설정 (CSP, X-Frame-Options 등)
- **모니터링:**
  - `Sentry` (에러 트래킹 및 성능 모니터링)
- **배포:** `Vercel`

## 7. 단계별 마일스톤 및 런칭 전략

**Step 1. POC & 핵심 가설 검증 (프로토타입 제작 / 2주)**
- **목표:** "코바늘 기호 이미지를 React-Konva 캔버스에 띄우고 부드럽게 스냅·이동시킬 수 있는가?"
- 더미 상태의 에디터 UI 구성, 5개 정도의 기초 기호 렌더링 테스트 진행.

**Step 2. MVP 개발 타겟팅 (1.5개월 ~ 2개월)**
- 회원가입/로그인 동선 적용 (Supabase Auth).
- 캔버스 상태(State)를 JSON 형식으로 DB에 CRUD(생성/읽기/수정/삭제) 하는 기능 구체화.
- [차트 $\rightarrow$ 서술형] 자동 번역 기초 룰셋 맵핑 및 알고리즘 구현.
- 브라우저 상의 결과물을 PDF로 다운받을 수 있는 기능 추가 (`jsPDF` + `html2canvas` 활용).

**Step 3. 비공개 베타 테스트 (CBT)**
- 뜨개 인스타그램 크리에이터나 관련 커뮤니티에서 **극 초기 사용성 테스터 20명** 모집.
- 실제 그들이 쓰던 도안을 에디터로 옮겨보게 하며 버그 점검, 특히 UX(단축키 등)의 불편함 해결.

**Step 4.Phase 1 정식 런칭 & 기능 고도화**
- 랜딩 페이지 제작, 사용 방법 영상(튜토리얼) 제공 후 오픈. Free Tier 요금제 중심으로 바이럴 유도.

## 8. 보안 설계 (Security Architecture)

### 8.1 인증 및 세션 관리
- **Supabase Auth** 기반 소셜 로그인 (Google, Apple) + 이메일/비밀번호
- 비밀번호 정책: 최소 8자, 영문+숫자+특수문자 조합 권장
- JWT Access Token (1시간 만료) + Refresh Token (7일 만료)
- 향후 2FA(이중 인증) 지원 (Phase 2 이후 검토)

### 8.2 데이터 접근 제어 (Row Level Security)
- Supabase RLS 정책을 통해 사용자는 **자신의 도안 데이터만** 조회/수정/삭제 가능
- 공유 도안의 경우 읽기 전용 접근 정책 별도 정의
- API Rate Limiting: 인증 사용자 기준 분당 60회, 비인증 요청 차단

### 8.3 파일 업로드 보안 (Phase 2)
- SVG 업로드 시 **DOMPurify** 기반 Sanitization 필수
- 허용 태그 화이트리스트: `<svg>`, `<path>`, `<circle>`, `<rect>`, `<line>`, `<polyline>`, `<polygon>`, `<g>`, `<defs>`, `<use>`
- `<script>`, `<foreignObject>`, 이벤트 핸들러 속성 (`onload`, `onclick` 등) 완전 제거
- 파일 크기 제한: SVG 최대 500KB

### 8.4 PDF 내보내기 보안
- 클라이언트사이드에서 `jsPDF` + `html2canvas`로 렌더링 (서버사이드 렌더링 미사용으로 SSRF 위험 차단)
- 사용자 입력 텍스트(도안 제목, 메모 등)는 PDF 생성 시 이스케이프 처리

### 8.5 개인정보 보호
- 전송 구간 암호화: HTTPS (TLS 1.2+) 필수
- 저장 데이터 암호화: Supabase 기본 제공 PostgreSQL 암호화 적용
- 개인정보처리방침 및 이용약관 페이지 제공
- 계정 삭제 시 사용자 데이터 30일 내 완전 삭제 (GDPR/개인정보보호법 대응)

### 8.6 에러 처리 및 복원력
- 네트워크 끊김 감지 시 사용자에게 오프라인 상태 알림 및 로컬 임시 저장
- 자동 저장 실패 시 재시도 큐 (최대 3회) 및 사용자 수동 저장 유도
- Sentry를 통한 프론트엔드/백엔드 에러 실시간 모니터링
